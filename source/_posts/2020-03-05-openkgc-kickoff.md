# 背景和动机

经过二三十年的互联网技术发展，我们积累了丰富的原始数据资源，但整体上数据开放、共享和流通力不足，数据有效利用呈现孤岛化，数据及应用相关责权不明确，迫切需要建立新型的的数据管理体系。

过去10年大数据处理技术不断成熟，使得那些拥有某领域大量数据的互联网厂商，基于全局视角对数据的洞察，让使用互联网媒介（PC & Mobile）的普罗大众大为受益。
对数据价值的纯熟利用，也使得传统的企业管理提高了投入产出比，获益良多。

虽然数据由每个互联网的参与个体产生，但是对个人数据的掌控并不能做到得心应手，由此引发了一系列隐私问题和社会困扰。
另一方面，信息丰富的应用平台利用自身优势，基于推荐算法向参与个体推送信息，不仅推荐算法的内部逻辑呈黑盒而难以做到公平（多是在特定平台的商业最大化原则驱动下调整算法），
而且在宏观层面上形成了对群体兴趣和行为的导向，进而扼杀了个体面向信息世界探索和创新力，这对一个生产力丰富的社会长期上是伤害巨大的。

<!-- more -->

从更长的时间尺度来看，我们依然处在对数据利用的初级阶段。这个初级阶段中，我们对数据定义更多局限在人和特定垂直领域服务的交互。一个生态系统具有以下过程特征：
**服务因满足特定需求而吸引众人，人与服务交互产生数据的积累，数据的规模效应反过来帮助服务体验的提升，进而人与服务产生更多的交互，以此产生良性循环**。

通过十多年来以来数据的积累、算力的提升、算法的智能化，我们实现了一个阶段性目标，即通过数据更好（而非完全）地了解众人。
虽然这不是当前阶段数据智能做到的所有事情，也算是大部分了。简而言之，**数据这一工具被一小部分企业使用，而让大多数人受益**。

随着5G网络、量子计算等新型基础设施的升级，以及未来一二十年新技术的不断涌现，我们将迈过当前的初级阶段，向增强型的数据智能高阶段迈进。
我们将有更强的算力、算法，更快的通信设施，覆盖更广的数据采集设备。
我们将能够对人、居家环境、城市、乃至周边世界进行数字化重建，让物理世界以另外一种方式展现在我们面前。

数字孪生，能够比较形象地描述这一未来景象。数字孪生的世界，并非虚拟游戏或架空历史描述的世界，一个虚无的、与现实独立存在的世界。
相反地，数字孪生世界和物理世界密不可分，是人类文明发展到一定阶段的客观反映。数字孪生世界来源于物理世界，而又作用于物理世界。
数据智能被普罗大众所掌握利用，像使用火、锤子、电一样，作为一种新的生产力工具，既改变着外在的物理世界，也深刻影响着我们每个人的生活和思维方式。

这便是我现在满怀期待、可能未来每个人司空见惯的世界。
数据智能这一工具不再是小部分企业的智力高地，而是能够走进千千万万普罗大众日常生活的新的生活资料。

人们在创造、掌握更多数据的同时，也应该能够享受到同等对数据利用的自由和权利。
在数字孪生的世界中，让数据智能造福众生。

# 开放知识计算

我于2020年初发起 [Open Knowledge Computing](https://github.com/OpenKGC)（开放知识计算）项目，众人拾薪来实现这一目标。
当然实现上述的愿景和目标并不容易，不仅需要创新先进的技术做基础，而且需要强有力的商业执行力保驾护航。
即使这一目标暂时过于遥远，也希望能够激发更多人对“数据智能造福众生”理念的认识，积极参与其中。

下面我将从技术上对知识计算系统中关键组件的一些思考。

# 知识计算相关技术

## 理论基础

作为知识计算的核心数据模型，超图结构具有支持原生逻辑推理技术的特点。OpenCog的Linas Vepstas和我持有类似的观点，
可以参考[Why Hypergraph](https://blog.opencog.org/2013/03/24/why-hypergraphs/)
在数学中，超图（Hypergraph）是一种广义上的图，它的一条边可以连接任意数量的顶点。
对表示现世界中概念的内在联系，例如表征事件及其内在参与角色，具有天然而直觉上的优势。
其挑战在于成熟的超图数据存储系统，以及基于超图模型的推理引擎的实现。

另外一个强有力的理论支撑领域是知识图谱，在当前人工智能的第4次浪潮力，知识图谱相关技术在深度网络和SOTA自然语言算法、工具的加持下，取得了诸多的进步。
知识图谱理论是个庞大的领域，尤其为知识图谱构建自动化贡献了不菲的成果。
但是众多研究者被诸多细小的问题隔离在一个个小格子里，追求极致的局部优化准确率，容易陷入“见木不见林”的狭义的成功满足之中。
总结起来，知识图谱包括以下活跃的子领域，在知识计算的征途上，需要主动吸收各子领域优秀的思想，集之大成。
* 关系预测 Relation Prediction (a.k.a. Link Prediction)
* 属性预测 Attribute Prediction
* 子图预测 Subgraph Prediction
* 概念嵌入 Building Concept Embeddings
* 规则挖掘 Rule Mining (a.k.a. Association Rule Learning)
* 本体合并 Ontology Merging
* 自动化知识图谱构建技术 Automated Knowledge Graph Creation
* 专家系统 Expert Systems
* 模式挖掘优化 Optimal Pattern Finding
* 系统自动化 System Design Automation and Configuration Automation
* 模糊匹配 Fuzzy Pattern Matching
* 自然语言交互 Querying and Responding via Natural Language

逻辑与推理理论，是在另一个近些年被统计机器学习技术（如深度神经网络、增强学习等）占了上风、对于知识计算却是强有力支撑的理论。
在前两次的人工智能浪潮中，逻辑推理技术尚受到重视。在知识计算中需要首先着力探索演绎、归纳、溯因的技术实现，
这里如果能充分系统统计机器学习领域的成果，便再好不过了。
当然也需要认清现实，真正的逻辑推理目前人类的理论段位尚不能达到，首先应该做到“形而上”的逻辑推理实现，再进而探索属于知识计算领域的“空气动力学”。
（笔者注：航空领域的发展并非来自鸟类翅膀的仿生学，而是得益于空气动力学理论的发现）。
* 演绎推理 Deduction：由前提已知的事实，确定地得到推理。
* 归纳推理 Induction：论证的前提支持结论但不确保结论的推理过程。
* 溯因推理 Abduction：开始于事实的集合，并推导出其最佳解释的推理过程。
